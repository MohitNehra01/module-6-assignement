1. to

    (Headings)
    • Use Cases:
      • Structuring content hierarchically
      • Creating headings and subheadings
    • Best Practices:
      • Use only one <h1> per page for the main title
      • Use headings in a logical order without skipping levels (e.g., <h1> followed by <h2>, not <h4>)

  2. (Paragraph)

    • Use Cases:
      • Wrapping blocks of text into paragraphs
    • Best Practices:
      • Use <p> for any significant block of text
      • Avoid using multiple <p> tags for a single logical paragraph
  3. (Anchor)
    • Use Cases:
      • Creating hyperlinks to other pages or sections within the same page
    • Best Practices:
      • Always include descriptive text within the <a> tag to improve accessibility
      • Use relative URLs for internal links and absolute URLs for external links
  4. (Strong Importance)
    • Use Cases:
      • Emphasizing important text
    • Best Practices:
      • Use <strong> to indicate strong importance, not just for styling
      • Avoid overusing to maintain its effectiveness
  5. (Emphasis)
    • Use Cases:
      • Emphasizing text in a subtle way, often for italicizing
    • Best Practices:
      • Use <em> to convey emphasis that changes the meaning of the sentence
      • Do not use <em> for decorative purposes

  6. (Line Break)
    • Use Cases:
      • Inserting a line break within text, such as in addresses or poems
    • Best Practices:
      • Use sparingly and only when a line break is necessary
      • Avoid using <br> to create spacing; use CSS for layout purposes
